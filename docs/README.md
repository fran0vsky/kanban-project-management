# Docs

Naming convention for easy recall:

| Pattern | Meaning | Examples |
|--------|--------|----------|
| **`spec-<topic>.md`** | What we're building (scope, features, roadmap) | `spec-project.md` |
| **`rules-<topic>.md`** | How we build (conventions by stack/area) | `rules-context.md`, `rules-angular.md`, `rules-nestjs.md` |

- All lowercase, hyphenated.
- Refer in chat or code as e.g. **spec-project**, **rules-angular**, **rules-nestjs**, **rules-context**.
